About this project

让无线信道模型在新场景中，只用少量样本就能快速适配。

本项目研究跨场景少样本无线信道建模：不再只依赖单一场景的大量标注数据，而是构建代表性源场景、学习可迁移信道表征，并通过支持样本驱动的类比适配完成目标场景快速建模。

核心动机是现实部署中的场景变化：传播环境、空间几何、反射/遮挡关系和 CSI 分布都会改变。如果每换一个场景都重新采集大量数据并完整训练模型，部署成本会很高。因此，本项目把问题拆成“源场景如何组织、表征如何预训练、目标少样本如何使用、适配策略如何自动选择”四个连续环节。

查看核心结果 理解方法框架
一句话贡献 一个面向低成本部署的 cross-scenario few-shot pipeline：代表性源场景构建 → CSI/CIR 双视图预训练 → support-driven analogical adaptation → lightweight strategy optimization。

Research question

这个项目解决什么问题？

01

无线场景变化很快

单场景训练的模型在新传播环境中容易失效，因为 CSI 分布、空间相关性和多径结构都会发生变化。

02

目标场景标注很少

实际部署时往往只有少量测量样本，直接 fine-tuning 容易过拟合，direct transfer 又缺乏局部适配能力。

03

源场景不是越多越好

不同源场景对目标任务的帮助不同，需要研究代表性源场景构建，而不是盲目聚合所有数据。

04

support 样本质量很关键

少样本适配不只取决于样本数量，还取决于 support set 是否覆盖目标区域的空间结构。

Step 1

代表性源场景构建

根据场景统计特征、几何传播特征与潜在表征线索组织源场景，目标是在有限 source budget 下保留跨场景覆盖能力，减少冗余源数据带来的训练成本。

source budgetscenario descriptorcross-scenario coverage

Core intuition

少样本不是只看“多少个点”，而是看这些点是否解释了目标区域。

实验表明，clustered support 持续最差；random / diverse / proposed 明显更好。这说明 support samples 的空间覆盖会直接影响最终适配质量。
